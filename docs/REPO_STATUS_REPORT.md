### 2.2. docs/VISION.md — Оценка: 8/10

Качественный стратегический документ. Чётко структурирован и покрывает:

| Раздел | Содержание | Качество |
|---|---|---|
| North Star | Уничтожение «информационной слепоты» | Ясно сформулировано |
| B2C-функции | Live Score, Semantic AI Search, AI Planner, Heat Map | Детально описаны проблемы и решения |
| B2B-функции | AI-Consulting, Spyglass, Direct Connect, The Hook | Конкретные бизнес-сценарии |
| Backlog | Offline Mode, Affiliate CPA | Правильно отложены за MVP |
| Технология | Google Gemini AI + социальные сигналы | Упоминается, но не конкретизировано |

**Сильные стороны документа:**
- Каждая функция описана через формулу «Проблема → Возможность → Результат»
- Чёткое разделение B2C и B2B
- Понятная отсечка MVP от будущих фич

**Слабые стороны документа:**
- Нет User Stories или Use Cases в формализованном виде
- Нет критериев приоритизации фич внутри MVP
- Не определён порядок реализации (что первое, что второе)
- Нет метрик успеха (KPI) для каждой функции
- Нет описания целевой аудитории (география, демография)

## 11. Рекомендации по приоритету

### Немедленные действия (до начала кодинга)

1. **Создать `.gitignore`** — защита от утечки секретов
2. **Выбрать и зафиксировать технологический стек** (ADR)
3. **Определить MVP-scope** — выбрать 2-3 ключевые фичи из Vision
4. **Спроектировать архитектуру** — API-дизайн, схема БД, структура проекта
5. **Настроить проектный scaffolding** — package.json, конфиги, линтеры

### Краткосрочные (первые 2 спринта)

6. **Настроить CI/CD** — GitHub Actions (lint + test + build)
7. **Создать базовый backend** — API endpoints для core-функций
8. **Создать базовый frontend** — UI-каркас с навигацией
9. **Настроить Docker** — для единообразия среды разработки
10. **Написать первые тесты** — задать культуру тестирования с самого начала

### Среднесрочные (месяц 1-2)

11. **Интегрировать Gemini AI** — начать с Semantic Search
12. **Реализовать Live Score** — ключевой дифференциатор продукта
13. **Настроить мониторинг** — логирование, error tracking

## 13. Заключение

LiveCity — это проект с **сильной продуктовой концепцией**, но находящийся на **нулевой стадии технической реализации**. Документ Vision демонстрирует глубокое понимание проблемы и рынка. Однако между текущим состоянием и работающим MVP лежит значительный объём инженерной работы.

**Главный вызов проекта:** перейти от идеи к коду, не потеряв фокус и не утонув в scope creep.

**Готовность к разработке:** Концептуально — да. Технически — нет. Необходимы архитектурные решения, выбор стека и определение MVP-scope.
